# Specification: MVP - Initial Version

**Track ID:** mvp_20260304
**Type:** Feature
**Created:** 2026-03-04
**Status:** Draft

## Summary

Build the initial version of My Own Todo App — a full-stack web app that manages tasks as Obsidian-compatible markdown files, indexed in SQLite, with cross-device sync via Obsidian Headless Sync.

## Context

Existing todo apps lock data in proprietary formats and don't integrate with Obsidian. This MVP delivers a dedicated task management web UI for Obsidian power users, with full data ownership via plain-text `.md` files.

## User Story

As an Obsidian power user, I want a web-based task management UI that reads and writes Obsidian-compatible markdown files so that I can manage my tasks with a polished interface while keeping full data ownership and cross-device sync.

## Acceptance Criteria

- [ ] Tasks are stored as Obsidian-compatible `.md` files with proper frontmatter and emoji task format
- [ ] SQLite index is built from vault files and supports full rebuild
- [ ] Web UI supports basic task CRUD — create, view, complete, and delete tasks
- [ ] Inbox quick-capture — add tasks to `Inbox/inbox.md` from the web UI
- [ ] Projects and Areas are browsable — list projects/areas and their tasks
- [ ] App runs in Docker alongside Obsidian Headless Sync for cross-device sync
- [ ] Daily Planning ritual — interactive daily task curation creating daily notes in `Calendar/`
- [ ] Authentication via NextAuth.js (credentials or passkey for solo user)
- [ ] Real-time file watching via Chokidar detects sync changes and reindexes
- [ ] Recurring tasks supported
- [ ] Drag-and-drop task reordering
- [ ] Search and filtering across tasks
- [ ] Tags/labels parsed and manageable in UI
- [ ] Mobile-responsive UI

## Dependencies

None

## Out of Scope

- Telegram integration
- Google Calendar integration

## Technical Notes

- **Obsidian Tasks emoji format**: Define supported emojis (📅 due, ✅ done, ⏫ priority, 🔁 recurring, etc.) for compatibility with the Obsidian Tasks plugin
- **Vault directory structure**: Establish conventions upfront (`Inbox/`, `Projects/`, `Areas/`, `Calendar/`)
- **Write conflict handling**: Both the web app and Obsidian Sync can write to `.md` files — adopt a strategy (last-write-wins + reindex after sync)
- **SQLite as derived cache**: All writes go to `.md` files first, then update SQLite. Never write to SQLite without writing the file first.
