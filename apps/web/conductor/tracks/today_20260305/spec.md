# Specification: Today — Daily Planning

**Track ID:** today_20260305
**Type:** Feature
**Created:** 2026-03-05
**Status:** Draft

## Summary

The "Today" feature is the heart of the app — a dual-mode daily planning view. **Planning Mode** lets the user curate a focused task list each morning (and re-plan throughout the day). **Focus Mode** is the clean, distraction-free execution view they live in after planning.

## Context

The product is a personal productivity web app that stores tasks as Obsidian-compatible markdown files. The daily plan creates a `Calendar/YYYY-MM-DD.md` daily note that references tasks from projects, areas, and inbox. SQLite provides fast queries, but the `.md` file is the source of truth.

## User Story

As a user, I want to plan my day each morning by curating a focused task list, and easily re-plan throughout the day as priorities shift, so that I always have a clear, actionable view of what matters today without losing track of deferred work.

## Acceptance Criteria

- [ ] Daily plan view is the default landing page — opening the app shows today's plan
- [ ] Plan creation ritual — user can curate today's plan by picking tasks from projects, areas, and inbox
- [ ] Re-plan at any time — user can add, remove, or reorder tasks throughout the day without friction
- [ ] Task completion updates source files — completing a task writes the done date back to the original `.md` file in Obsidian Tasks format
- [ ] Unfinished tasks are visible next day — incomplete tasks carry forward and are surfaced during the next planning session
- [ ] Daily note generated in `Calendar/` — each day's plan creates/updates an Obsidian-compatible daily note markdown file

## Dependencies

- Existing task model and SQLite database schema
- Markdown file parser and serializer (task-line parser, Obsidian Tasks format)
- Vault service (file read/write operations)
- Existing project/area/inbox structure and queries

## Out of Scope

- Weekly/monthly calendar views — this track is only about "Today"
- Time blocking / scheduling — no assigning specific times to tasks
- Smart suggestions / auto-prioritization — user manually curates the plan
- Notifications or reminders
- Productivity analytics — no streaks, completion rates, or dashboards
- Recurring task generation — handled by the `task-recurrence` track

## Technical Notes

- `/today` as the default route — app root (`/`) redirects to `/today`
- Daily note as source of truth — plan stored as `Calendar/YYYY-MM-DD.md` in Obsidian daily note format
- Task references, not copies — daily note references tasks from source files via identifiers
- `scheduledDate` field (⏳ emoji) marks tasks as planned for a given date
- Server Actions for mutations — add-to-plan, remove-from-plan, reorder, complete-task, finish-planning
- Carry-forward query — SQLite query for tasks scheduled for past dates never marked done
- Optimistic UI for reordering — instant feel with server action confirmation
- Dual-mode UI — Planning Mode for curation, Focus Mode for execution; mode tracked via daily note frontmatter (`planned: true/false`)
