# Specification: People Management

**Track ID:** people-management_20260304
**Type:** Feature
**Created:** 2026-03-04
**Status:** Draft

## Summary

Link Obsidian `[[@Person Name]]` references in tasks to a people index, enabling a contacts directory and per-person task views. Add an `@` mention autocomplete when editing tasks.

## Context

The app stores tasks as Obsidian-compatible markdown, parsed by a task-line parser and indexed into SQLite. People references (`[[@Person Name]]`) are a standard Obsidian wiki-link convention but are not currently parsed or indexed. People notes live in a `People/` vault folder.

## User Story

As an Obsidian power user, I want to link people to tasks so I can track commitments and follow-ups per person.

## Acceptance Criteria

- [ ] Parse `[[@Person Name]]` references from task lines and store them in a people/mentions table in SQLite
- [ ] Auto-discover people from vault files in the `People/` folder (with frontmatter metadata)
- [ ] People directory page (`/people`) listing all known contacts with count of linked tasks
- [ ] Person detail view (`/people/[slug]`) showing all tasks mentioning that person, grouped by project/area
- [ ] `@` mention autocomplete UI when typing task titles — suggests known people and inserts `[[@Name]]`

## Dependencies

- Existing task-line parser (`src/lib/markdown/task-line-parser.ts`)
- SQLite indexer (`src/db/indexer.ts`)
- Drizzle schema (`src/db/schema.ts`)
- File serializer (`src/lib/markdown/file-serializer.ts`)

## Out of Scope

- Email, messaging, or calendar integrations
- Visual relationship/network graphs between people

## Technical Notes

- Use Obsidian `[[@Person Name]]` wiki-link format for references
- People notes expected in `People/` vault folder with optional frontmatter (e.g., email, company)
- Many-to-many relationship: tasks can mention multiple people, people can appear in many tasks (use a `task_people` join table)
- `@` autocomplete queries the `people` SQLite table for suggestions
