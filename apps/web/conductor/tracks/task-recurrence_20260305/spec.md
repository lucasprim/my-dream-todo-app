# Specification: Task Recurrence Patterns

**Track ID:** task-recurrence_20260305
**Type:** Feature
**Created:** 2026-03-05
**Status:** Draft

## Summary

Enhance the existing recurrence system to support full calendar recurrence patterns (simple and complex) and provide a UI for setting/editing recurrence rules on tasks.

## Context

The app already has basic recurrence infrastructure: the `🔁` emoji format, a simple `nextRecurrenceDate()` parser in `src/lib/recurrence.ts`, and completion logic that generates the next occurrence. However, only simple patterns are supported (every day/week/month/year, every N units, every weekday, every `<day>`). Popular todo apps like Things 3, Todoist, and Obsidian Tasks support much richer patterns.

## User Story

As an Obsidian power user, I want to set rich recurrence patterns on tasks so that completed recurring tasks automatically generate the next occurrence with the correct due date.

## Acceptance Criteria

- [ ] Simple recurrence patterns — Support daily, weekly, monthly, and yearly recurrences (e.g., "every day", "every week", "every 3 months")
- [ ] Complex recurrence patterns — Support advanced rules like "every 2nd Tuesday", "every last weekday of month", "every Mon/Wed/Fri"
- [ ] Next occurrence generation — When a recurring task is completed, the next occurrence is automatically created with the correct due date
- [ ] Obsidian-compatible format — Recurrence data stored in the task line using `🔁` emoji notation, compatible with Obsidian Tasks plugin
- [ ] Recurrence UI — Users can set/edit recurrence patterns through the web UI when creating or editing a task

## Dependencies

- Existing task-line parser/serializer (`src/lib/markdown/task-line-parser.ts`, `task-line-serializer.ts`)
- Existing recurrence module (`src/lib/recurrence.ts`)
- SQLite schema `recurrence` column (`src/db/schema.ts`)
- Task completion server action (`src/app/actions/task-actions-impl.ts`)
- Existing UI task editing components

## Out of Scope

- Recurring task history/log (tracking all past occurrences)
- Calendar view for recurring tasks
- Bulk editing recurrence on multiple tasks
- Notification/reminders for recurring tasks

## Technical Notes

- Follow best practices from Todoist/Things 3 for recurrence UX (preset buttons for common patterns, natural language display)
- Use `rrule` library (RFC 5545) for complex pattern calculation, storing human-readable text in markdown but using RRULE internally for date computation
- Support "after completion" vs "on fixed schedule" distinction (e.g., Todoist's "every day" vs "every! day")
