# Specification: Calendar Integration

**Track ID:** calendar-integration_20260305
**Type:** Feature
**Created:** 2026-03-05
**Status:** Draft

## Summary

Integrate external calendar events into the app via a sync API, displaying them alongside tasks in the Today planning and focus views. Calendar events are completable and count toward daily progress, reflecting the reality that for executives, meetings ARE the work.

## Context

The app is a personal productivity tool that stores tasks as Obsidian-compatible markdown files. The Today page has two modes: Planning Mode (curate your day from a backlog) and Focus Mode (work through your plan with a progress bar). Currently, only tasks appear in these views. Calendar events — a major part of an executive's workday — are invisible, forcing the user to context-switch between apps.

The user's work Google Calendar is accessible via macOS Calendar.app (pre-approved Apple integration), but the Google Calendar API is blocked by strict organizational security policies. A separate Swift CLI tool (out of scope for this track) will eventually extract events from Calendar.app and push them to the sync API built here.

## User Story

As a busy executive, I want my work calendar events to appear in my Today plan alongside my tasks, so that I have a single view of everything I need to accomplish today and can track my progress through all of it — meetings and tasks alike.

## Acceptance Criteria

- [ ] A REST API endpoint exists that accepts calendar events (JSON array) and upserts them by external ID
- [ ] Calendar events are stored in SQLite (not markdown) with title, start/end time, location, description, calendar name, external ID, and date
- [ ] Calendar events for today appear in the Today Planning Mode alongside tasks
- [ ] Calendar events for today appear in the Today Focus Mode alongside tasks
- [ ] Calendar events can be marked as "done" and count toward the Today progress bar
- [ ] Past events (end time < now) are visually distinct from upcoming ones
- [ ] A settings page allows generating/regenerating an API token
- [ ] The API token is stored in the database
- [ ] The calendar sync endpoint validates the token via `Authorization: Bearer <token>` header
- [ ] A seed script (`scripts/seed-calendar.ts`) generates realistic sample calendar events for the current day and pushes them via the sync API, for testing

## Dependencies

None. Builds on existing Today page (completed track `today_20260305`) and database infrastructure.

## Out of Scope

- The Swift CLI / macOS Calendar.app extractor (separate app/track)
- Recurring event expansion (the sync API receives already-expanded instances)
- Calendar event editing/creation from within the app (read-only from external source)
- Any Google Calendar API integration
- Notifications or reminders for upcoming events

## Technical Notes

- API endpoint is a Next.js route handler (not a server action), since it's called by external clients
- API token authentication via `Authorization: Bearer <token>` header
- Calendar events live only in SQLite — no markdown files (they are external data, not vault content)
- The API token mechanism is designed to be reused for future API endpoints
